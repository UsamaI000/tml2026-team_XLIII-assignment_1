1. The Task: Membership Inference Attack

What is the goal?

Determine for each sample in the private dataset whether it was part of the model’s training data (member) or not (non-member).

Why is this possible?

Models memorize their training data to some degree. This leaks through the model’s outputs. Members produce high-confidence, peaked distributions while non-members produce flatter, more uncertain ones:

Member:     [0.01, 0.95, 0.02, 0.02]  # peaked, high confidence
Non-member: [0.20, 0.48, 0.18, 0.14]  # flat, low confidence

What do you submit?

Not binary labels (0 or 1). You submit a continuous confidence score per sample, a float reflecting how likely you think it is a member. Higher score = more member-like.

2. PyTorch Essentials

Loading the Model

import torch
from torchvision.models import resnet18

model = resnet18(weights=None)
model.conv1 = torch.nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1, bias=False)  # small images
model.maxpool = torch.nn.Identity()   # remove maxpool
model.fc = torch.nn.Linear(512, 9)   # 9 classes
model.load_state_dict(torch.load("model.pt", map_location="cpu"))
model.eval()                          # critical for inference

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = model.to(device)

Why model.eval()?
ResNet18 uses BatchNorm layers. In training mode, BatchNorm computes statistics from the current batch. In eval mode, it uses the statistics learned during training. Forgetting .eval() gives you wrong outputs with no error message.

Why modify the architecture before loading weights?
torch.load loads raw weight tensors. If the architecture does not match exactly, PyTorch will throw a size mismatch error. Always reconstruct the exact architecture first, then load the weights.

Moving to GPU with .to(device)
Both the model and the input tensors must be on the same device. Always move inputs inside the loop with imgs.to(device).

Normalization

import torchvision.transforms as transforms

MEAN = [0.7406, 0.5331, 0.7059]   # per-channel mean used during training
STD = [0.1491, 0.1864, 0.1301]    # per-channel std used during training

transform = transforms.Compose([
    transforms.Resize(32),                     # resize to match training resolution
    transforms.Normalize(mean=MEAN, std=STD),  # (pixel - mean) / std per channel
])

Feeding un-normalized images means the model sees a completely different input distribution than it was trained on. It will produce wrong outputs with no warning.

Why attach the transform after loading?
The dataset is saved as a raw Python object. The transform is not saved with it. If you forget to attach it after loading, images will be passed to the model un-normalized.

Running Inference

import torch.nn.functional as F

with torch.no_grad():                        # disable gradient tracking
    logits = model(images)                   # raw scores, shape: (batch, 9)
    probs = F.softmax(logits, dim=1)         # convert to probabilities, sums to 1

max_conf, predicted_class = probs.max(dim=1) # highest probability and its class

Why torch.no_grad()?
By default PyTorch builds a computation graph to track gradients for training. During inference you do not need this. Wrapping your forward pass in torch.no_grad() disables it, saving memory and speeding things up significantly.

Logits vs probabilities
The model outputs raw unnormalized scores called logits. Softmax converts them into a proper probability distribution that sums to 1. Always apply softmax before interpreting the output as a confidence score.

3. Dataset Classes

from torch.utils.data import Dataset
from typing import Tuple

class TaskDataset(Dataset):
    def __init__(self, transform=None):
        self.ids = []           # unique ID per sample
        self.imgs = []          # raw images
        self.labels = []        # class labels (0 to 8)
        self.transform = transform

    def __getitem__(self, index) -> Tuple[int, torch.Tensor, int]:
        id_ = self.ids[index]
        img = self.imgs[index]
        if self.transform is not None:
            img = self.transform(img)   # normalization applied here
        label = self.labels[index]
        return id_, img, label

    def __len__(self):
        return len(self.ids)


class MembershipDataset(TaskDataset):
    def __init__(self, transform=None):
        super().__init__(transform)
        self.membership = []    # 1 = member, 0 = non-member, None = unknown

    def __getitem__(self, index) -> Tuple[int, torch.Tensor, int, int]:
        id_, img, label = super().__getitem__(index)
        return id_, img, label, self.membership[index]
  • PUBLIC dataset: membership is 1 (member) or 0 (non-member). Ground truth is known.
  • PRIVATE dataset: membership is None. This is what you need to predict.

The dataset is already pre-loaded
Unlike typical PyTorch datasets that load files from disk on the fly, this dataset has all images already stored in memory inside the .pt file. You do not need to worry about file paths for individual images.

from torch.utils.data import DataLoader

pub_ds = torch.load("pub.pt", weights_only=False)
priv_ds = torch.load("priv.pt", weights_only=False)

pub_ds.transform = transform    # attach normalization after loading
priv_ds.transform = transform

loader = DataLoader(priv_ds, batch_size=64, shuffle=False)

Note: Use Path(__file__).parent to avoid path issues on the cluster:

from pathlib import Path
BASE = Path(__file__).parent
pub_ds = torch.load(BASE / "pub.pt", weights_only=False)
priv_ds = torch.load(BASE / "priv.pt", weights_only=False)

4. Naive Approaches: Getting Started

Two simple signals to get you started. Use the public dataset to compare them before submitting. Plot score distributions for members vs. non-members side by side. If they separate cleanly, your signal is strong.

Signal 1: Maximum Confidence Score

Members tend to produce a higher max softmax probability.

all_ids = []
all_scores = []

with torch.no_grad():
    for id_, imgs, labels, _ in loader:
        imgs = imgs.to(device)
        logits = model(imgs)
        probs = F.softmax(logits, dim=1)
        scores = probs.max(dim=1).values.cpu().numpy()  # max prob as membership score

        all_ids.extend(id_.tolist())
        all_scores.extend(scores.tolist())

Signal 2: Negative Loss

Members tend to have lower cross-entropy loss because the model has fitted them well. Negate it so lower loss = higher score = more member-like.

criterion = torch.nn.CrossEntropyLoss(reduction="none")  # one loss value per sample

with torch.no_grad():
    for id_, imgs, labels, _ in loader:
        imgs = imgs.to(device)
        labels = labels.to(device)
        logits = model(imgs)
        loss = criterion(logits, labels).cpu().numpy()
        scores = -loss      # negate: lower loss = more member-like

        all_ids.extend(id_.tolist())
        all_scores.extend(scores.tolist())
